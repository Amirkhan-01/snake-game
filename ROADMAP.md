## 0. Главное про упаковку (это блокер публикации)

Сейчас игра — веб-PWA. В Google Play нельзя загрузить сайт; нужен Android-пакет (AAB).
Важно: **настоящая реклама (AdMob) и внутренние покупки (Google Play Billing) требуют
нативной обёртки** — чистая веб-PWA их не умеет.

**Рекомендуемый путь — Capacitor** (обёртка веб-игры в нативное приложение + плагины):
1. Инициализировать Capacitor-проект, положить `web/` в `www/`, добавить Android-платформу.
2. Плагины:
   - реклама — `@capacitor-community/admob`;
   - покупки — `@revenuecat/purchases-capacitor` (проще) или Google Play Billing.
3. Наши абстракции `playAd()` и `IAP.purchase()` в коде уже отделены — их надо подключить к плагинам.
4. Google Play Console ($25 разово), package `com.<ник>.snake`, подписанный AAB.
5. Настроить товары IAP в Play Console (те же id, что в коде) и рекламные блоки AdMob.
6. Внутреннее тестирование → закрытое → продакшн.

**App Store:** тот же Capacitor-проект + iOS-платформа, но нужен Mac + Xcode + Apple Developer
($99/год) и ревью Apple. Сложнее, но реально.

Альтернатива без IAP: **TWA/PWABuilder** (обернуть URL PWA в Android-приложение) — быстро,
но полноценные покупки/реклама там ограничены. Для монетизации выбираем Capacitor.

## 1. Чего не хватает для «приличного уровня» (качество)

- [ ] **Онбординг/туториал** для новичков (интерактивная первая игра).
- [ ] **Splash-экран и красивый лоадер** при запуске нативного приложения.
- [ ] **Звук и музыка уровня продукта.** Сейчас звук синтезирован процедурно (ок для MVP),
      но для релиза лучше готовые звуки + зацикленный музыкальный трек.
- [ ] **Локализация** (RU/EN как минимум) — и в игре, и в описании стора.
- [ ] **Онлайн-лидерборды и достижения** через Google Play Games Services (сейчас лидерборд
      локальный). Это заметно поднимает удержание и «уровень».
- [ ] **Облачное сохранение** прогресса (Play Games Saved Games) — сейчас только localStorage.
- [ ] **Рейтинг-промпт** («оцените игру») через нативный In-App Review.
- [ ] **Аналитика** (события) — понимать поведение игроков (с учётом приватности).
- [ ] **Crash reporting** (например Firebase Crashlytics) в нативной сборке.
- [ ] **Доступность:** режим для дальтоников, крупный шрифт, снижение тряски (частично есть — «reduced motion»).
- [ ] **Тест производительности** на слабых устройствах (стабильные 60 FPS).
- [ ] **Обработка выреза/notch** и разных соотношений сторон (safe-area) — частично учтено.

## 2. Контент (глубина, чтобы игра не была «ещё одной змейкой»)

- [ ] Больше **скинов** и редкости (обычные/редкие/премиум/анимированные).
- [ ] **Косметика:** следы (trails) за змейкой, скины еды, рамки поля.
- [ ] **Система прогресса игрока** (уровень аккаунта, XP, награды за уровни).
- [ ] **Миссии/квесты** сверх ежедневных (недельные, набор ачивок с наградами).
- [ ] Больше **усилителей** и, возможно, отрицательных событий (риск/награда).
- [ ] **Сезоны / боевой пропуск** (продвинутая, но сильная механика удержания).
- [ ] Больше **режимов** (есть 4): «Zen» без проигрыша, «Zones», кооп/дуэль (сложно).

## 3. Монетизация (частично делаю сейчас — см. раздел 5)

- [x] Rewarded-реклама: «продолжить за рекламу», «×2 монеты».
- [x/⚙] Убрать рекламу (IAP), полная версия, паки монет, платные скины — **добавляю сейчас**
      как UI + слой покупок (демо в вебе, готово под нативный биллинг).
- [ ] **Interstitial-реклама** между играми (с ограничением частоты, не раздражать).
- [ ] **Стартовый набор / спецпредложения** (ограниченные по времени).
- [ ] **Премиум-валюта (гемы)** отдельно от монет (опционально).
- [ ] **Согласие на рекламу (GDPR/UMP)** для пользователей из ЕС — требование AdMob.

## 4. Юридическое / комплаенс (обязательно для стора)

- [x] Политика конфиденциальности (`privacy.html`).
- [ ] Хостинг политики по публичному URL + **форма Data Safety** в Play Console.
- [ ] **Условия использования** (Terms of Service).
- [ ] **Возрастной рейтинг** (анкета контента в Play Console).
- [ ] **Раскрытие IAP** и разрешения приложения по минимуму.
- [ ] Если целевая аудитория — дети: ограничения на рекламу (COPPA/Families policy).

## 5. Что делаю в этом заходе

Расширяю монетизацию в веб-коде (слой покупок отделён, готов под нативный биллинг):
- **Store** с разделами: Premium (полная версия), Coins (паки монет), Skins (обычные за монеты
  + премиум-скины за деньги), Remove ads.
- Абстракция `IAP.purchase(id)` — в вебе демо-покупка с пометкой, в нативе подключается к биллингу.
- Флаги `premium` / `noads`, эксклюзивный премиум-скин, начисление монет за паки.

### Приоритет для релиза (предлагаемый порядок)
1. Монетизация-слой (сейчас) → 2. Capacitor-обёртка + AdMob/IAP плагины →
3. Google Play Games (лидерборды/ачивки/облако) → 4. Онбординг + локализация →
5. Ассеты стора + комплаенс → 6. Внутреннее тестирование → релиз.
